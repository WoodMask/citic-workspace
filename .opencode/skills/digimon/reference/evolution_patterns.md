# Evolution Patterns Reference

Digimon 技能进化模式参考。

## 进化模式分类

### 1. 结构优化模式

| 模式 | 适用场景 | 操作 |
|------|----------|------|
| **章节重组** | 流程描述分散 | 合并相关章节，优化顺序 |
| **层级调整** | 章节层级混乱 | 统一 ## 和 ### 使用 |
| **表格优化** | 表格信息冗余 | 简化表格，合并相似行 |
| **代码块整理** | 示例代码过多 | 分类整理，添加注释 |

### 2. 指令优化模式

| 模式 | 适用场景 | 操作 |
|------|----------|------|
| **触发简化** | 触发条件复杂 | 简化表述，优先列核心条件 |
| **流程可视化** | 流程描述冗长 | 添加流程图/ASCII图 |
| **约束强化** | 约束描述模糊 | 明确约束值，添加检查方法 |
| **示例补充** | 缺少使用示例 | 添加典型使用示例 |

### 3. 内容优化模式

| 模式 | 适用场景 | 操作 |
|------|----------|------|
| **冗余删除** | 内容重复 | 删除重复描述 |
| **空白优化** | 空行过多 | 压缩连续空行 |
| **表达精简** | 指令过长 | 拆分长句，简化表达 |
| **术语统一** | 术语不一致 | 统一关键词使用 |

### Path 3: 约束强化

```
原版:
## 约束规则
每次进化的变体必须通过以下约束：
- 测试通过
- 大小限制
- 语义一致

进化版:
## 约束规则

> ⚠️ 所有约束为**强制检查项**，必须全部通过才能应用变体。

| 约束 | 规则 | 检查方法 |
|------|------|----------|

| **语义保留** | 不偏离原始目的 | DSPy semantic check |
| **破坏性检查** | 无删除关键章节 | diff 分析 |
```

**改进点**: 明确约束值，添加检查方法，强调重要性

## 进化度量指标

### 质量指标

| 指标 | 计算方式 | 目标值 |
|------|----------|--------|
| **准确率** | 测试通过率 | ≥ 80% |
| **清晰度** | 平均行长度评分 | ≥ 0.75 |
| **效率** | 独特行占比 | ≥ 0.9 |
| **完整性** | 关键章节覆盖率 | ≥ 100% |

## 进化失败模式

### 常见失败原因

| 失败类型 | 原因 | 解决方案 |
|----------|------|----------|
| **约束失败** | 变体超过大小限制 | 减少变异率，精简内容 |
| **语义偏离** | 删除关键章节 | 强化结构保留约束 |
| **评分过低** | 变体质量下降 | 增加反思迭代，调整变异策略 |
| **无改进** | Pareto 前沿空 | 放宽约束，增加评估数据 |

### 回滚策略

当进化失败时：

```markdown
<!-- Digimon Rollback: 2026-04-23 | reason: constraint_failed -->

## 回滚记录

**失败变体**: variant-003
**失败原因**: 大小超过 15KB 限制
**回滚至**: backup_20260423_103000.md
```

## 高级进化技巧

### 1. 批量进化

同时进化多个相关 skill：

```python
skills_to_evolve = [
    'open-source-tech-report',
    'send-citic-mail',
    'citic-workboard'
]

for skill in skills_to_evolve:
    evolver = SkillEvolver(skill, iterations=10)
    result = evolver.evolve()
```

### 2. 条件触发进化

基于使用频率触发：

```python
if skill_usage_count(skill_name) > threshold:
    trigger_evolution(skill_name)
```

### 3. 协同进化

让 self-improving-agent 和 digimon 协同：

```
self-improving-agent 收集失败经验
        ↓
存储到 memory/episodic/
        ↓
digimon 读取经验作为评估数据
        ↓
执行进化优化
        ↓
应用改进版本
```

## 最佳实践总结

1. **从小迭代开始**: 先 5 次迭代验证效果
2. **使用真实数据**: sessiondb 比 synthetic 更有效
3. **启用反思**: 反思功能显著提升进化质量
4. **保留历史**: 备份原版本，记录进化历史
5. **用户确认**: 大改动需用户确认后应用
6. **定期清理**: 清理旧变体历史避免膨胀
